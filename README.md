# 🎲 Яцзи (Yatzy) — Telegram Mini App

Multiplayer Yatzy для двух игроков, интегрированный как Telegram Mini App.

## Стек

| Слой | Технология |
|------|-----------|
| Frontend | React + TypeScript + Vite |
| Backend | Node.js + Express + Socket.io |
| Протокол | WebSocket (real-time) |
| Интеграция | Telegram Web App SDK |

## Быстрый старт

### Установка зависимостей

```bash
# Сервер
cd server && npm install

# Клиент  
cd client && npm install
```

### Запуск в разработке

**Сервер** (порт 3001):
```bash
cd server
npm run dev
```

**Клиент** (порт 5173):
```bash
cd client
npm run dev
```

Открой http://localhost:5173 в браузере.

## Деплой для Telegram Mini App

### Сервер
Задеплой на любой VPS / Railway / Render и запомни URL (например `https://yatzy.example.com`).

### Клиент
1. Обнови `VITE_SERVER_URL` в `client/.env`:
   ```
   VITE_SERVER_URL=https://yatzy.example.com
   ```
2. Собери:
   ```bash
   cd client && npm run build
   ```
3. Залей папку `dist/` на Vercel / Netlify / Nginx.

### Telegram Bot
1. Создай бота через [@BotFather](https://t.me/BotFather)
2. Введи `/newapp` → укажи URL твоего задеплоенного клиента
3. Поделись ссылкой на Mini App с другом

## Игровые правила

**Ходы:** каждый игрок бросает кубики до 3 раз за ход, удерживая нужные.  
**Очки:** выбери категорию для записи очков после бросков.

### Подсчёт очков

| Секция | Категория | Очки |
|:-------|:----------|:-----|
| **Верхняя** | Единицы … Шестёрки | Сумма кубиков с выбранной гранью |
| | **Бонус** | **+35** (если сумма верхней секции ≥ 63) |
| **Нижняя** | Три одинаковых | Сумма всех кубиков |
| | Четыре одинаковых | Сумма всех кубиков |
| | Фулл-хаус | 25 |
| | Малая улица | 30 (4 последовательных значения) |
| | Большая улица | 40 (5 последовательных значений) |
| | Яцзи (Yatzy) | 50 (5 одинаковых) |
| | Шанс | Сумма всех кубиков |

### Lucky Streak Multiplier (LSC)

В игре реализована механика комбо для **Нижней секции**:
- При выборе категории из Нижней секции несколько раз подряд, применяется множитель очков.
- Базовый множитель — **x1.0**.
- Каждое последовательное заполнение категории в Нижней секции увеличивает множитель на **+0.2** (максимум **x2.0**).
- **Исключение:** Категория **Шанс (Chance)** не участвует в накоплении комбо, на неё не действует множитель, и её выбор сбрасывает стрик.
- **Сброс:** Стрик сбрасывается до **x1.0** в следующих случаях:
  - Выбор категории из **Верхней секции**.
  - Выбор категории **Шанс**.
  - Если выбранная категория в Нижней секции закрывается на **0 очков** (вычеркивание).

### Yatzy Bonus (Повторный Яцзи)

Если вы выбрасываете Яцзи (5 одинаковых), а категория **Яцзи** у вас уже заполнена на **50 очков**:
- Вы можете закрыть на **100 очков** либо категорию в **Верхней секции**, соответствующую номиналу кубиков (например, "Четвёрки", если выпало 5 четвёрок), либо **любую** свободную категорию в **Нижней секции** (3 одинаковых, 4 одинаковых, Фулл-хаус, Улицы, Шанс).
- **Важно:** Множитель LSC **не применяется** к этим 100 очкам (вы всегда получаете ровно 100).
- При выборе категории из Нижней секции, она **продолжает стрик** (увеличивает ваш множитель для следующих ходов).
- Если вы выберете несовпадающую категорию в Верхней секции (например, "Единицы" при 5 четвёрках), вы получите за неё **0 очков** (стандартное правило сброса).
- Бонус не активируется, если первая категория Яцзи была закрыта на 0.


*Пример: Первый Фулл-хаус даст 25 очков (x1.0), следующая за ним Малая улица даст 36 очков (30 * 1.2). Если после этого закрыть Яцзи на 0, стрик сбросится.*

## Структура проекта

```
yatzy-app/
├── client/              # React + Vite Frontend
│   ├── src/
│   │   ├── components/  # Die, ScoreCard, GameBoard, Lobby, GameOver
│   │   ├── hooks/       # useSocket, useTelegram
│   │   ├── types/       # game.ts
│   │   ├── utils/       # yatzy.ts (score calc)
│   │   ├── App.tsx
│   │   └── main.tsx
│   └── index.html       # Telegram Web App SDK подключён здесь
│
└── server/              # Node.js + Express + Socket.io Backend
    └── src/
        ├── game/
        │   └── yatzyLogic.ts
        ├── index.ts     # Socket events, room management
        └── types.ts
```

## Socket API

| Событие (client → server) | Данные | Описание |
|----------------------------|--------|----------|
| `create_room` | `{ name }` | Создать комнату |
| `join_room` | `{ code, name }` | Войти в комнату |
| `roll_dice` | — | Бросить кубики |
| `toggle_hold` | `{ index }` | Зафиксировать/снять кубик |
| `score_category` | `{ category }` | Записать очки |
| `rematch` | — | Реванш |

| Событие (server → client) | Описание |
|---------------------------|----------|
| `room_created` | Комната создана, код |
| `game_started` | Второй игрок подключился |
| `game_state` | Обновление состояния игры |
| `game_over` | Игра окончена, победитель |
| `player_disconnected` | Противник отключился |
| `error` | Ошибка |
