# План внедрения системы эмодзи-реакций (Reactions Implementation Plan)

Этот план описывает шаги по добавлению всплывающих эмодзи-реакций в игру Yatzy.

## 1. Серверная часть (Backend)
- [x] **Регулирование событий в `server/src/index.ts`**:
  - Добавить лимит времени отправки реакций (rate limiting): 1 реакция в 2.5 секунды на сокет.
  - Создать обработчик события `send_reaction` с валидацией эмодзи по белому списку: `['👍', '🔥', '🎉', '💩', '😎', '😢']`.
  - Пересылать реакцию оппоненту через событие `receive_reaction` с полезной нагрузкой `{ senderId: string, emoji: string }` (используя `socket.to(code).emit`).

## 2. Клиентская часть: Интеграция сокетов и хуки
- [x] **Обновление `client/src/hooks/useSocket.ts`**:
  - Добавить функцию `sendReaction(emoji: string)` для отправки события `send_reaction`.
  - Зарегистрировать слушатель `receive_reaction` и передавать полученные реакции в коллбек компонента `GameBoard`.

## 3. Клиентская часть: Логика локальной игры (Бот)
- [x] **Интеграция ИИ-реакций в `client/src/hooks/useLocalGame.ts`** (перенесено на уровень контроллера GameBoard для оптимизации):
  - Создать поддержку симуляции реакций от бота. Если игрок отправляет эмодзи в игре с ботом, бот с вероятностью 70% должен ответить взаимностью через 1–2 секунды (выбрав подходящий эмодзи).
  - Добавить автоматические реакции бота при его ходах: эмодзи `🔥` или `😎` при выпадении Yatzy, и эмодзи `😢` при закрытии категории на 0 очков.

## 4. Клиентская часть: Верстка и Стили (UI/UX)
- [x] **Добавление CSS-анимаций в `client/src/components/GameBoard.css`**:
  - Создать стили для плавающего эмодзи `.floating-reaction` с абсолютным позиционированием и `pointer-events: none`.
  - Реализовать keyframe-анимацию `@keyframes floatUpReaction`, которая плавно поднимает эмодзи вверх, раскачивает его по горизонтали, увеличивает масштаб до `1.5x` и уводит прозрачность в `0`.
- [x] **Разработка интерфейса панели реакций в `client/src/components/GameBoard.tsx`**:
  - Добавить кнопку-триггер `💬` в заголовок `board-header` возле счета игрока.
  - Создать состояние `activeReactions` (массив объектов `{ id, emoji, senderId }`) для рендеринга активных анимаций.
  - Создать всплывающую панель выбора реакций (Tray), автоматически закрывающуюся по таймеру или при клике вне её.
  - Добавить генерацию случайного смещения (X/Y) для каждой летящей реакции, чтобы при частой отправке эмодзи не слипались.
  - Добавить вызов haptic feedback при отправке и получении реакции (через `useTelegram`).

## 5. Тестирование и приемка
- [x] Собрать и проверить проект на предмет ошибок компиляции TypeScript/Vite (`npm run build` успешно завершен).
- [x] Проверить отправку реакций в режиме игры против бота.
- [x] Убедиться, что бот адекватно реагирует на эмодзи игрока.
- [x] Запустить локальный сервер и клиент, проверить мультиплеерное прохождение реакций между двумя вкладками браузера.
